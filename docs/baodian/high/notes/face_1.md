## 单页与多页应用有什么区别？

::: tip 参考
首先我们先区分开两者的区别：

**单页面应用（SPA）**

时下流行的一种应用模式，只加载一个 html 文件，页面切换其实只是切换组件（页面片段），实际都是在这个页面内部通过动态加载组件来实现页面跳转的效果。从实现上来讲，主要是监听 url 变化，通过路由来动态加载或卸载某些组件。
- 优点
  - 页面切换快，响应更加迅速
  - 相对来说对服务器压力小，因为内容的改变不需要重新加载整个页面
  - 前后端分离，职责更加明确
- 缺点
  - 首屏加载速度慢
  - 不利于 SEO
  - 抛开易上手的 vue 来讲，其他的框架学习成本较高

**多页面应用**

多页面应用更倾向于传统的开发模式，
:::
